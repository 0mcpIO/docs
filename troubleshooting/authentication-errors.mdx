---
title: Authentication errors
description: Fix 401, 403, and missing credential errors.
---

Authentication errors usually mean the upstream API did not receive the value it expects.

## Quick checks

1. Open **Setup** and compare every required header and environment variable with the client configuration.
2. Confirm a Bearer value includes the `Bearer ` prefix when required.
3. Check the API key header name, including spelling and dashes.
4. Replace an expired or revoked token.
5. Confirm the credential can access the selected API operation.
6. Reconnect the MCP client after changing an environment variable.

For a query API key, use the gateway header generated by 0mcp. Do not add the secret directly to the endpoint URL.

A `401` usually means a credential is missing or invalid. A `403` usually means the credential is known but does not have permission.
