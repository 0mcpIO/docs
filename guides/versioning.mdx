---
title: Versioning
description: Publish changes safely and return to an older setup.
---

A version is a saved copy of your server's tools, resources, and prompts. The live MCP endpoint always uses one published version.

## Publish changes

Make your edits, then open **Versions**. Review the capability counts and validation notes. Publish the new version when it is ready.

Connected clients see the new capability list after they reconnect or refresh tools. Existing calls already in progress continue with the setup they started with.

## Return to an older version

Open **Versions**, choose a known good version, and publish it. This makes its saved capabilities live again. Your newer draft and version history remain available.

Pausing is separate from versions. Pause serving when you need to stop all calls without changing which version is published.

<Tip>Add a short note to each version so you can tell why it was created.</Tip>
