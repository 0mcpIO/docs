---
title: Analytics
description: Understand usage and performance numbers.
---

Analytics shows how clients use your server. Open a server dashboard or **Activity** to review the available totals and trends.

Common values include:

- Calls by tool, resource, or prompt
- Successful and failed calls
- Response time
- Request and response data size
- MCP client usage

Use the date and capability filters to narrow the view. A rise in errors may point to an expired credential, changed upstream API, invalid input, or paused server. Start with [Authentication errors](/troubleshooting/authentication-errors) for `401` or `403` results. A rise in response time may point to the upstream API or a large response.

Analytics stores safe usage details. It does not need the credential or full payload to count a call. Use [Logs](/guides/logs) when you need to narrow the result to a specific time or capability.
