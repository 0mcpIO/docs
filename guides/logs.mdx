title: Logs
description: Review calls and changes without exposing secrets.
---

Open **Activity** to see operational activity and audit events.

Operational entries help you review calls. They can include the capability name, result, response time, data size, and time. Audit events show changes such as imports, edits, publishing, pausing, and domain updates.

## Find an issue

1. Set a time range close to when the issue happened.
2. Filter by capability type, name, status, or MCP client.
3. Open the matching entry and note its result and time.
4. Compare it with the upstream API logs for the same period.

0mcp keeps log data safe by leaving out credentials and request payloads. If you need the exact upstream request body, use your own API logs with suitable secret filtering.
